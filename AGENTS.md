## Cursor Cloud specific instructions

GlinTech BOM Insight is a **frontend-only** single-page/SSR app (TanStack Start + React 19 + Vite 8, Tailwind v4). It uses **Bun** as the package manager (`bun.lock`, `bunfig.toml`); there is no backend, database, or external service — all data comes from `src/lib/mock-data.ts`. The UI is Hebrew (RTL).

- **Package manager:** Use `bun`, not npm/yarn. `bun` is installed on `PATH` (also symlinked at `/usr/local/bin/bun`). The startup update script runs `bun install`.
- **Dev server:** `bun run dev` serves on `http://localhost:8080/` (port/host are fixed by `@lovable.dev/vite-tanstack-config`, not the standard Vite 5173).
- **Build:** `bun run build` (production SSR build via nitro). For a dev-mode build use `bun run build:dev`.
- **Lint:** `bun run lint`. Note: the committed source currently has many pre-existing `prettier/prettier` lint errors, so `lint` exits non-zero out of the box — this is not an environment problem. `bun run format` (prettier --write) would fix them, but do not reformat existing files unless asked.
- **Routing:** File-based via `src/routes/`. `src/routeTree.gen.ts` is auto-generated by the dev server/build and may show up as a local diff — do not hand-edit or commit it.
- **Config plugin:** `vite.config.ts` relies on `@lovable.dev/vite-tanstack-config`, which already bundles the React/Tailwind/TanStack/path-alias plugins — do not add them manually.
